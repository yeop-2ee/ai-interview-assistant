## 1. User 테이블

> 서비스 사용자(일반/관리자) 계정 정보를 저장합니다.

| 컬럼명 | 데이터 타입 | 제약조건 | 기본값 | 설명 |
|--------|-----------|---------|--------|------|
| id | Int | PK, AUTO_INCREMENT | - | 사용자 고유 ID |
| name | String | NOT NULL | - | 사용자 이름 |
| email | String | NOT NULL, UNIQUE | - | 이메일 주소 (로그인 ID) |
| password | String | NOT NULL | - | bcrypt 해싱된 비밀번호 |
| role | String | NOT NULL | `"user"` | 권한 (`user` / `admin`) |
| createdAt | DateTime | NOT NULL | `now()` | 계정 생성일시 |

### 인덱스

| 인덱스명 | 컬럼 | 유형 |
|---------|------|------|
| User_email_key | email | UNIQUE |
| User_pkey | id | PRIMARY KEY |

### 관계

- `User` 1 : N `InterviewReport` (onDelete: Cascade)

## 2. InterviewReport 테이블

> 면접 완료 후 생성되는 AI 분석 결과 리포트를 저장합니다.

| 컬럼명 | 데이터 타입 | 제약조건 | 기본값 | 설명 |
|--------|-----------|---------|--------|------|
| id | Int | PK, AUTO_INCREMENT | - | 리포트 고유 ID |
| userId | Int | NOT NULL, FK(User.id) | - | 리포트 소유자 ID |
| title | String | NOT NULL | `"면접 결과"` | 리포트 제목 (직무명 포함) |
| overallScore | Int | NOT NULL | `0` | 종합 점수 (0~100) |
| scores | Json | NOT NULL | `{}` | 영역별 점수 JSON |
| questions | Json | NOT NULL | `[]` | 질문별 상세 피드백 JSON 배열 |
| fullReport | Json | NULLABLE | `null` | 전체 리포트 원본 JSON |
| createdAt | DateTime | NOT NULL | `now()` | 리포트 생성일시 |

### scores JSON 구조

```json
{
  "answerContent": 85,
  "logicStructure": 78,
  "delivery": 80,
  "reliability": 82,
  "likability": 76
}
```

### questions JSON 구조 (배열)

```json
[
  {
    "question": "자기소개 해주세요",
    "answer": "안녕하세요...",
    "category": "공통",
    "followup": null,
    "feedback": {
      "adequacy": "적절",
      "lengthEval": "적당",
      "improvedAnswer": "개선 예시...",
      "expectedFollowup": "예상 꼬리질문..."
    }
  }
]
```

### 인덱스

| 인덱스명 | 컬럼 | 유형 |
|---------|------|------|
| InterviewReport_pkey | id | PRIMARY KEY |
| InterviewReport_userId_fkey | userId | FOREIGN KEY |

## 3. KnowledgeEntry 테이블

> 관리자가 등록하는 학과별 전공지식 DB입니다. 면접 질문 생성 시 참고 자료로 활용됩니다.

| 컬럼명 | 데이터 타입 | 제약조건 | 기본값 | 설명 |
|--------|-----------|---------|--------|------|
| id | Int | PK, AUTO_INCREMENT | - | 항목 고유 ID |
| department | String | NOT NULL | - | 학과명 (예: 컴퓨터소프트웨어과) |
| subject | String | NOT NULL | - | 과목명 (예: 자료구조) |
| content | String | NOT NULL | - | 전공 지식 내용 (주요 개념, 예상 질문 등) |
| registeredBy | String | NOT NULL | `""` | 등록한 관리자 이름 |
| createdAt | DateTime | NOT NULL | `now()` | 등록일시 |

### 인덱스

| 인덱스명 | 컬럼 | 유형 |
|---------|------|------|
| KnowledgeEntry_pkey | id | PRIMARY KEY |

### 조회 패턴

- `WHERE department = ?` — 학과별 전체 조회
- `ORDER BY createdAt ASC` — 등록순 정렬
- `TAKE 10` — 질문 생성 시 최신 10개만 활용
